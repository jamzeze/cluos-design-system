# CluOS Patterns

Esta pasta contém especificações detalhadas de padrões de interface para todos os sistemas CluOS.

## Arquivos

| Arquivo | Conteúdo |
|---------|----------|
| [components.md](./components.md) | Especificação de cada componente: objetivo, anatomia, tokens, estados, acessibilidade |
| [page-templates.md](./page-templates.md) | Templates de página reutilizáveis para fluxos comuns |
| [states.md](./states.md) | Especificação completa de estados de interface |
| [accessibility.md](./accessibility.md) | Requisitos e padrões de acessibilidade |
| [content-guidelines.md](./content-guidelines.md) | Linguagem, tom, vocabulário e escrita de interface |

## Como usar

1. Antes de criar uma tela, verificar se há um template adequado em [page-templates.md](./page-templates.md).
2. Usar os componentes especificados em [components.md](./components.md) com os tokens oficiais.
3. Garantir que todos os estados definidos em [states.md](./states.md) estejam implementados.
4. Verificar acessibilidade com o checklist em [accessibility.md](./accessibility.md).
5. Revisar o texto com as diretrizes em [content-guidelines.md](./content-guidelines.md).
